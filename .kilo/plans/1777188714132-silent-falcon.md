# AICL Frontier Proposals Implementation Plan

## Objective
Establish a formal "frontier ideation" track within the AICL repository to capture high-impact, speculative research proposals without contaminating the canonical kernel or relying on excessive marketing hype.

## Guidelines
- **Anti-Hype Policy:** Eradicate terms like "definitive AI-native language", "post-human computing", or "magical guarantees". Use sober, academically rigorous terminology.
- **Kernel Isolation:** No changes will be made to the `docs/kernel/` directory. All work will reside in `docs/research/`.
- **Classification:** Treat these proposals as experimental candidates requiring rigorous benchmarks and formal circuit/model design before acceptance.

## Execution Steps

1. **Create Research Directory Structure**
   - Create `docs/research/shg-tensor/`
   - Create `docs/research/machine-autonomy/`
   - Create `docs/research/neural-symbolic-bridge/`
   - Create `docs/research/autopoietic-runtime/`

2. **Draft Frontier Proposals Index**
   - File: `docs/research/FRONTIER_PROPOSALS_INDEX.md`
   - Content: An index of speculative, high-value research tracks with explicit caveats regarding their experimental nature and evaluation criteria.

3. **Draft Holographic IR Candidate Encoding**
   - File: `docs/research/shg-tensor/HIR_CANDIDATE_ENCODING.md`
   - Focus: Vector Symbolic Architecture (VSA) / Hyperdimensional Computing (HDC) as an experimental encoding for SHG, not a finalized "machine-native syntax".

4. **Draft zk-Intent Subtrack**
   - File: `docs/research/machine-autonomy/ZK_INTENT_DRAFT.md`
   - Focus: Cryptographic proofs of compiled artifacts or verification circuits, explicitly noting that SNARK/STARK circuit design is the primary bottleneck and it does not magically solve broad semantic compliance.

5. **Draft Continuous Semantic Relaxation**
   - File: `docs/research/neural-symbolic-bridge/CONTINUOUS_SEMANTIC_RELAXATION_DRAFT.md`
   - Focus: Differentiable projection generating candidate optimizations, with the strict boundary rule that it cannot satisfy Tier 1 proof obligations until snapped back to symbolic SHG form.

6. **Draft Autopoietic Telemetry Binding**
   - Files: `docs/research/autopoietic-runtime/README.md` and `docs/research/autopoietic-runtime/AUTOPOIETIC_TELEMETRY_BINDING_DRAFT.md`
   - Focus: Runtime semantic feedback and bounded hyperedge repair. Replace "living code" with precise terminology.
