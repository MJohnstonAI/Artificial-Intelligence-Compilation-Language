# Recommended GitHub Labels

These labels are recommended for managing public review issues.

They do not affect AICL semantics.

## Review Labels

- `ai-evaluation` — structured evaluation generated by or with an AI model
- `contradiction` — conflicting claims or semantic inconsistency
- `challenge-brief` — test brief for ICC, SHG, proof, materializer, or ROM behavior
- `public-review` — issue created during public technical review

## Specification Labels

- `kernel` — canonical Kernel v0.1.1 contract layer
- `semantic-layer` — detailed semantic support under `docs/kernel/semantic/`
- `wkg` — World Knowledge Graph grounding, anchors, snapshots, or ontology
- `proof-tier` — Tier 1 / Tier 2 / Tier 3 proof classification
- `shg` — Semantic Hypergraph schema or behavior
- `haig` — Human-AI Integration Gate
- `pact` — Policy-Aware Coordination Treaty
- `materializer` — target artifact emission and preservation contracts
- `rom` — Residual Obligation Manifest

## Research Labels

- `research-track` — non-canonical experimental proposal
- `shg-tensor` — SHG-TENSOR representation or benchmarks
- `machine-icc` — MachineICC autonomous intent contracts
- `neural-symbolic` — neural-symbolic bridge research
- `meta-circularity` — grammar/type-system self-representation research
- `scope-control` — issue related to preventing scope explosion

## Implementation Labels

- `implementation` — future executable toolchain work
- `validator` — ICC/SHG/WKG/proof artifact validation
- `parser` — parsing or grammar-related implementation
- `good-first-review` — suitable for first-time reviewers
- `spec-quality` — clarity, formatting, terminology, or organization improvement

## Priority Labels

- `critical`
- `high`
- `medium`
- `low`
