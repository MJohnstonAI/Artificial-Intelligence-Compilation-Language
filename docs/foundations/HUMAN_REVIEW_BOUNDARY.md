# Human Review Boundary

Human review is welcome. It should be bounded by AICL's design objective: AI-native semantic compilation for AI systems, with human-readable projections used for inspection, governance, and critique.

## Useful Human Contributions

- Running AI model evaluations against the repository.
- Submitting contradiction reports.
- Identifying underspecified semantics.
- Submitting challenge briefs.
- Identifying missing related work.
- Testing whether example briefs can be compiled into ICC/SHG artifacts.
- Reviewing whether claims are overstated.

## Less Useful Contributions

- Asking AICL to become Python-like.
- Treating human syntax preference as the primary design objective.
- Reducing AICL to a prompt template.
- Reducing AICL to a conventional DSL.
- Dismissing semantic compilation because legacy languages can implement similar runtime behavior.
- Proposing changes that erase the AI-native premise.

## Reviewer Rule

The reviewer should first restate AICL's own design objective before criticizing it. Critique should evaluate whether AICL succeeds on its own terms, not whether it conforms to legacy programming-language expectations.

A useful critique says: given AICL's stated goal of AI-native semantic compilation, this type authority rule is incomplete, this proof obligation is mislabeled, this operational transition is underspecified, or this claim overstates the current implementation status.

A less useful critique says: this should become more like a familiar human-first language because that language is already comfortable to write.
